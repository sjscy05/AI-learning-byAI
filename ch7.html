📊 第七章：无监督学习与聚类

整理衣柜 —— 让机器自己发现规律

想象你有一大堆乱七八糟的衣服，你想把它们整理好。你不需要提前定义"什么是上衣、什么是裤子"，而是让机器自动把相似的东西放在一起。这就是无监督学习。

无监督学习：没有标签的训练数据，让算法自己发现数据中的结构和规律。核心挑战在于没有"正确答案"来指导学习。

无监督学习的三大任务：

任务目标典型算法应用场景
聚类将数据分组成有意义的簇K-Means、DBSCAN、层次聚类客户分群、图像分割
降维减少特征数量保留核心信息PCA、t-SNE、UMAP数据可视化、特征压缩
异常检测识别偏离正常模式的数据点Isolation Forest、LOF欺诈检测、故障诊断

K-Means聚类 —— 最简单的聚类算法

生活比喻：想象你是一个学校食堂管理员，需要决定开几个打饭窗口。你会观察学生们的排队习惯，自动把他们分成几组。

K-Means的工作步骤：

  1. 初始化：随机选择K个数据点作为初始中心
  2. 分配：将每个数据点分配到最近的中心点
  3. 更新：重新计算每个簇的中心点
  4. 重复：直到中心点不再变化或达到最大迭代次数

K-Means的数学表达：

K-Means的目标是最小化簇内平方误差（Within-Cluster Sum of Squares, WCSS）：

其中 $\mu_k$ 是第 $k$ 个簇的中心点，$r_{nk}$ 是指示变量（$r_{nk}=1$ 表示点 $n$ 属于簇 $k$）。

K-Means的收敛性：

定理：K-Means算法在每次迭代中都会使目标函数 $J$ 单调递减，因此保证收敛。但由于目标函数是非凸的，算法可能收敛到局部最优解。

收敛过程保证：

  • 分配步骤：每个点分配到最近的中心，使 $J$ 减小或不变
  • 更新步骤：中心点移向簇的均值，使 $J$ 减小或不变
  • $J$ 有下界（$\geq 0$），且每次迭代严格递减

因此，K-Means一定在有限步内收敛。通常10-30次迭代即可收敛。

如何选择K值？—— 肘部法则（Elbow Method）

import numpy as np
from sklearn.cluster import KMeans
import matplotlib.pyplot as plt

# 生成示例数据
np.random.seed(42)
X = np.vstack([
    np.random.randn(100, 2) * 0.8 + [0, 0],
    np.random.randn(100, 2) * 0.8 + [4, 4],
    np.random.randn(100, 2) * 0.8 + [0, 4]
])

# 肘部法则：计算不同K值的WCSS
wcss = []
K_range = range(1, 11)
for k in K_range:
    kmeans = KMeans(n_clusters=k, random_state=42, n_init=10)
    kmeans.fit(X)
    wcss.append(kmeans.inertia_)  # inertia_ 就是WCSS

# 绘制肘部图
plt.figure(figsize=(8, 4))
plt.plot(K_range, wcss, 'bo-')
plt.xlabel('K (簇数)')
plt.ylabel('WCSS (簇内平方误差)')
plt.title('肘部法则选择最优K值')
plt.grid(True)
plt.show()
# 选择"肘部"对应的K值（曲线下降速度突然变缓的点）

# 2. 对初始中心敏感（K-Means++改进）
# 3. 只能发现球形簇
# 4. 对异常值敏感

DBSCAN —— 基于密度的聚类

K-Means的局限性：只能发现球形簇，对噪声敏感。DBSCAN（Density-Based Spatial Clustering of Applications with Noise）解决了这些问题。

核心概念：

概念定义生活类比
ε-邻域以点为中心、半径为ε的圆形区域你家周围的"步行可达"范围
MinPts成为核心点所需的最少邻居数开饭馆需要的最少顾客数
核心点ε-邻域内包含至少MinPts个点的点热闹的十字路口
边界点不是核心点，但在某个核心点的ε-邻域内十字路口旁边的便利店
噪声点既不是核心点也不是边界点的点荒郊野外的孤零零房子

DBSCAN的工作流程：

  1. 选择任意一个未访问的点
  2. 计算其ε-邻域内的点数
  3. 如果是核心点（≥MinPts），创建一个新簇
  4. 从该核心点出发，将所有密度可达的点加入同一簇
  5. 如果遇到边界点，将其加入当前簇但不再向外扩展
  6. 重复以上步骤，直到所有点都被访问
from sklearn.cluster import DBSCAN
from sklearn.datasets import make_moons
import numpy as np

# 生成非球形数据（月牙形）
X, y_true = make_moons(n_samples=300, noise=0.08, random_state=42)

# K-Means无法正确聚类非球形数据
from sklearn.cluster import KMeans
kmeans_labels = KMeans(n_clusters=2, random_state=42).fit_predict(X)

# DBSCAN可以正确处理
dbscan = DBSCAN(eps=0.2, min_samples=5)
dbscan_labels = dbscan.fit_predict(X)

# 查看结果
n_clusters = len(set(dbscan_labels)) - (1 if -1 in dbscan_labels else 0)
n_noise = list(dbscan_labels).count(-1)
print(f"DBSCAN发现 {n_clusters} 个簇")
print(f"噪声点数量: {n_noise}")

# DBSCAN的优缺点：
# 优点：不需要指定K、能发现任意形状的簇、能识别噪声
# 缺点：对ε和MinPts敏感、高维数据效果差、密度不均匀时效果差

PCA降维 —— 数学推导

生活比喻：你在描述一个人时，可以说身高、体重、年龄、学历、收入等100个特征。但其实核心可能就是"经济实力"和"健康状况"两个维度。PCA就是帮你找到最重要的维度。

PCA的核心思想：找到数据变化最大的方向，保留最重要的信息，去掉冗余。

PCA的数学推导（特征值分解）：

给定 $n$ 个 $d$ 维样本 $\mathbf{x}_1, \mathbf{x}_2, \ldots, \mathbf{x}_n$，PCA的目标是找到 $k$ 个方向（$k < d$），使数据投影后的方差最大。

步骤1：中心化数据

步骤2：计算协方差矩阵

步骤3：特征值分解

步骤4：选择主成分

对特征值从大到小排序 $\lambda_1 \geq \lambda_2 \geq \cdots \geq \lambda_d$，选择前 $k$ 个特征向量，使得累积方差贡献率达到阈值（如95%）：

步骤5：投影

import numpy as np
from sklearn.decomposition import PCA

# 生成高维数据
np.random.seed(42)
X = np.random.randn(200, 10)  # 200个样本，10个特征

# PCA降维
pca = PCA()  # 不指定维度，保留所有成分
X_pca = pca.fit_transform(X)

# 查看各主成分的方差贡献率
print("各主成分方差贡献率:")
for i, ratio in enumerate(pca.explained_variance_ratio_):
    print(f"  PC{i+1}: {ratio:.4f}")

# 选择保留95%方差的维度
pca_95 = PCA(n_components=0.95)
X_reduced = pca_95.fit_transform(X)
print(f"\n原始维度: {X.shape[1]}")
print(f"保留95%方差后维度: {X_reduced.shape[1]}")

# 手动实现PCA（理解原理）
def my_pca(X, n_components):
    # 1. 中心化
    X_centered = X - np.mean(X, axis=0)
    # 2. 计算协方差矩阵
    cov = np.cov(X_centered.T)
    # 3. 特征值分解
    eigenvalues, eigenvectors = np.linalg.eig(cov)
    # 4. 按特征值排序
    idx = np.argsort(eigenvalues)[::-1]
    eigenvalues = eigenvalues[idx]
    eigenvectors = eigenvectors[:, idx]
    # 5. 选择前k个特征向量
    W = eigenvectors[:, :n_components]
    # 6. 投影
    X_projected = X_centered @ W
    return X_projected, eigenvalues

X_proj, eigenvals = my_pca(X, n_components=3)
print(f"\n手动PCA结果形状: {X_proj.shape}")

聚类评估指标

如何评价聚类结果的好坏？有两类指标：内部指标（不需要真实标签）和外部指标（需要真实标签）。

1. 轮廓系数（Silhouette Cohouette）

衡量每个样本的簇内紧密度与簇间分离度的平衡：

其中 $a(i)$ 是点 $i$ 到同簇其他点的平均距离，$b(i)$ 是点 $i$ 到最近其他簇的平均距离。轮廓系数范围 $[-1, 1]$，越接近1越好。

2. Calinski-Harabasz指数

也称为方差比准则，衡量簇间离散度与簇内离散度的比值：

其中 $B_k$ 是簇间离散度矩阵的迹，$W_k$ 是簇内离散度矩阵的迹。CH值越大，聚类效果越好。

3. 轮廓系数与CH指数对比：

指标计算复杂度范围越大越好适用场景
轮廓系数O(n²)[-1, 1]是中小规模数据
CH指数O(nkd)[0, +∞)是大规模数据
WCSSO(nkd)[0, +∞)否（越小越好）肘部法则选K
 
from sklearn.cluster import KMeans, DBSCAN
from sklearn.metrics import silhouette_score, calinski_harabasz_score
from sklearn.datasets import make_blobs
import numpy as np

# 生成示例数据
X, y_true = make_blobs(n_samples=300, centers=4, random_state=42)

# 尝试不同的K值
for k in range(2, 7):
    kmeans = KMeans(n_clusters=k, random_state=42, n_init=10)
    labels = kmeans.fit_predict(X)

    sil_score = silhouette_score(X, labels)
    ch_score = calinski_harabasz_score(X, labels)

    print(f"K={k}: 轮廓系数={sil_score:.4f}, CH指数={ch_score:.1f}")

# DBSCAN评估
dbscan = DBSCAN(eps=0.5, min_samples=5)
db_labels = dbscan.fit_predict(X)

# DBSCAN可能将部分点标记为噪声(-1)
mask = db_labels != -1
if len(set(db_labels[mask])) > 1:
    sil_db = silhouette_score(X[mask], db_labels[mask])
    ch_db = calinski_harabasz_score(X[mask], db_labels[mask])
    print(f"\nDBSCAN: 轮廓系数={sil_db:.4f}, CH指数={ch_db:.1f}")
    print(f"噪声点: {np.sum(db_labels == -1)}")

练习题

练习1：使用肘部法则确定最优K值。在Iris数据集上运行K-Means，绘制WCSS随K变化的曲线，找到最优K值。

💡 指引： K-Means假设簇是凸形且各向同性的。肘部法则的核心是观察WCSS下降速率的"拐点"——当K增加不再带来显著改善时，就接近最优值。思考：如果数据的真实簇形状不规则，K-Means还适用吗？

练习2：比较K-Means和DBSCAN在以下数据集上的表现：(a) 球形簇 (b) 月牙形簇 (c) 环形簇。分析各自的优缺点。

💡 指引： K-Means基于距离度量，倾向于发现球形簇；DBSCAN基于密度连通性，能发现任意形状的簇。考虑两种算法的核心假设：K-Means假设簇中心存在且大小相近，DBSCAN假设簇的密度均匀且有明确边界。

练习3：手动实现PCA（不使用sklearn），在2D数据上验证结果。将数据投影到方差最大的方向，可视化投影前后的数据。

💡 指引： PCA的关键步骤：中心化数据 → 计算协方差矩阵 → 特征值分解 → 选取最大特征值对应的特征向量作为投影方向。想想为什么投影方向就是方差最大的方向——这与特征向量的几何意义有关。

练习4：在客户分群场景中，使用轮廓系数和CH指数选择最优的聚类算法和参数。

💡 指引： 轮廓系数衡量簇内紧密度和簇间分离度的平衡（范围[-1,1]，越大越好）；CH指数衡量簇间距离与簇内方差的比值（越大越好）。注意：两者对不同簇形状的偏好不同，实际中应结合业务理解综合判断。
