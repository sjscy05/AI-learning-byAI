🤖 第一章：什么是AI？

教小孩认猫 —— 传统编程 vs 机器学习

想象你有一个3岁的小孩，你想教他认识什么是猫。传统编程的方式是这样：

传统编程：写一堆规则 —— "如果有尖耳朵、有胡须、有毛、会喵喵叫，那就是猫。"

但问题来了：世界上有各种各样的猫 —— 有的没有尾巴（波斯猫），有的很胖看不清耳朵，有的不叫只打呼噜。你需要写的规则会越来越多，最终无法穷举。

机器学习的方式完全不同：

机器学习：给小孩看1000张猫的照片和1000张不是猫的照片，让他自己总结规律。

这就是AI的核心思想：从数据中自动学习规律，而不是人工编写规则。

AI发展的三次浪潮

人工智能并非一帆风顺，而是经历了跌宕起伏的发展历程：

第一次浪潮（1956–1974）：黄金年代
1956年达特茅斯会议，"人工智能"正式诞生。人们充满乐观，认为20年内就能创造出通用智能。 代表性成果：ELIZA聊天机器人（能模拟对话）、感知机（最早的神经网络模型）。 但计算能力有限，理论逐渐遇到瓶颈。
第一次AI寒冬（1974–1980）
政府削减资金，研究陷入低谷。感知机无法解决异或（XOR）等非线性问题，批评声四起。
第二次浪潮（1980–1993）：专家系统
专家系统兴起：将领域专家的知识编码成规则。代表系统：MYCIN（医疗诊断）、XCON（计算机配置）。 反向传播算法让多层神经网络成为可能，但硬件仍然昂贵。
第二次AI寒冬（1993–2011）
专家系统维护成本过高，规则库越来越难管理。研究资金再次收缩。 但这一时期机器学习理论（SVM、决策树等）持续发展，为下一次突破积蓄力量。
第三次浪潮（2011–至今）：深度学习革命
2012年，AlexNet在ImageNet比赛中碾压传统方法；2016年，AlphaGo击败围棋世界冠军； 2017年，Transformer论文《Attention Is All You Need》问世； 2022年，ChatGPT发布，大语言模型进入公众视野。 GPU算力爆发 + 海量数据 + 算法突破，三者共同推动了这场革命。

机器学习的三大范式

机器学习是当前AI最核心的技术路径，根据学习方式的不同，可分为三大范式：

1. 监督学习（Supervised Learning） —— 有老师的学习

训练数据带有"正确答案"（标签），模型学习输入到输出的映射。

任务类型输出生活例子技术例子
回归连续数值预测明天的气温房价预测、股票价格
分类离散类别邮件是否是垃圾邮件人脸识别、疾病诊断

2. 无监督学习（Unsupervised Learning） —— 自学成才

训练数据没有标签，模型自行发现数据中的结构和规律。

任务类型目标生活例子技术例子
聚类将相似数据分组电商平台用户分群K-Means、DBSCAN
降维压缩数据保留核心信息从高维数据中找到关键因素PCA、t-SNE
异常检测找出不符合规律的数据信用卡欺诈检测Isolation Forest

3. 强化学习（Reinforcement Learning） —— 在试错中学习

智能体在环境中行动，根据获得的奖励/惩罚调整策略。核心概念：

  • 状态（State）：当前环境的情况（如棋盘局面）
  • 动作（Action）：智能体可以采取的行为（如落子位置）
  • 奖励（Reward）：环境反馈的即时反馈（如赢棋+1，输棋-1）
  • 策略（Policy）：根据状态选择动作的规则（智能体要优化的目标）

典型应用：AlphaGo（围棋）、机器人控制、游戏AI（Atari、星际争霸）。

机器学习的工作流程

一个完整的机器学习项目通常包含以下9个步骤：

  1. 问题定义 —— 明确要解决什么问题，评估指标是什么
  2. 数据收集 —— 获取相关的训练数据
  3. 数据清洗 —— 处理缺失值、异常值、重复数据
  4. 特征工程 —— 提取、选择、构造有用的特征
  5. 模型选择 —— 根据问题类型选择合适的算法
  6. 模型训练 —— 用训练数据调整模型参数
  7. 模型评估 —— 在验证集/测试集上评估模型表现
  8. 模型调优 —— 调整超参数，优化模型性能
  9. 部署上线 —— 将模型集成到实际系统中
💡 实际项目中，这9个步骤往往需要多次迭代。你可能会回到前面的步骤重新调整，直到模型达到满意的性能。

特征工程的完整流程

一个完整的特征工程流程包含四个核心步骤：

步骤1：数据清洗

数据质量决定模型效果的天花板。

import pandas as pd
import numpy as np

df = pd.read_csv('watermelon.csv')

# 缺失值处理
df.fillna(df.mean(), inplace=True)          # 数值型: 用均值填充
df['颜色'].fillna(df['颜色'].mode()[0])      # 类别型: 用众数填充

# 异常值检测 (IQR方法)
Q1 = df['重量'].quantile(0.25)
Q3 = df['重量'].quantile(0.75)
IQR = Q3 - Q1
df = df[(df['重量'] >= Q1-1.5*IQR) & (df['重量'] <= Q3+1.5*IQR)]

# 重复值删除
df.drop_duplicates(inplace=True)

步骤2：特征构造

从已有特征中创造更有信息量的新特征。

# 组合特征: 重量与大小的比值反映密度
df['密度'] = df['重量'] / (df['大小'] ** 3)

# 多项式特征: 捕捉非线性关系
from sklearn.preprocessing import PolynomialFeatures
poly = PolynomialFeatures(degree=2)
X_poly = poly.fit_transform(df[['重量', '大小']])

# 时间特征: 从日期中提取信息
df['购买日期'] = pd.to_datetime(df['购买日期'])
df['星期'] = df['购买日期'].dt.dayofweek
df['是否周末'] = df['星期'].isin([5, 6]).astype(int)

步骤3：特征选择

从众多特征中选出最有价值的子集。

方法原理优点适用场景
过滤法根据统计指标（相关系数、卡方检验）排序计算快、与模型无关高维数据快速筛选
包裹法用模型性能评估特征子集针对特定模型优化特征数量适中时
嵌入法模型训练中自动选择效率高、效果好树模型、正则化模型
 
# 过滤法: 相关系数
correlations = df.corr()['好瓜'].abs().sort_values(ascending=False)

# 嵌入法: L1正则化自动选择

步骤4：特征降维

当特征维度太高时，降维可以减少计算量、去除噪声。

# PCA: 保留95%方差
from sklearn.decomposition import PCA
pca = PCA(n_components=0.95)

t-SNE非线性降维，保留局部结构，不可逆数据可视化
UMAP比t-SNE更快，更好保留全局结构大规模数据可视化

特征工程的最佳实践

"数据和特征决定了机器学习的上限，而模型和算法只是逼近这个上限。" —— Andrew Ng

一个简单的模型 + 好的特征，往往比复杂模型 + 差的特征效果更好。以下是5条经过验证的最佳实践：

  • 先理解业务再动手 —— 特征工程不是盲目尝试，先理解数据背后的业务逻辑
  • 数据清洗优先 —— 垃圾进垃圾出，确保数据质量后再做特征构造
  • 从简单开始 —— 先用基础特征建基线模型，再逐步增加复杂度
  • 避免数据泄露 —— 特征缩放、缺失值填充等操作必须在训练集上fit，再应用到测试集
  • 特征可解释性 —— 可解释的特征更易调试、更值得信任

什么是监督学习？

监督学习就像有老师批改作业的学习方式：你做题（模型预测），老师告诉你对错（损失函数），你根据反馈调整（梯度下降）。

关键特点：训练数据有标签 —— 每个输入都对应一个正确的输出。

    def __init__(self, learning_rate=0.01, n_iterations=1000):
        self.lr = learning_rate
        self.n_iter = n_iterations
        self.weights = None
        self.bias = None
        self.losses = []

    def fit(self, X, y):
        n_samples, n_features = X.shape
        self.weights = np.zeros(n_features)
        self.bias = 0

        for _ in range(self.n_iter):
            y_pred = X.dot(self.weights) + self.bias
            loss = np.mean((y_pred - y) ** 2)
            self.losses.append(loss)

            dw = (1 / n_samples) * (2 * X.T.dot(y_pred - y))
            db = (1 / n_samples) * (2 * np.sum(y_pred - y))

            self.weights -= self.lr * dw
            self.bias -= self.lr * db

    def predict(self, X):
        return X.dot(self.weights) + self.bias

激活函数详解

激活函数是神经网络的灵魂。不同的激活函数有不同的数学特性和适用场景：

1. Sigmoid 函数

  • 输出范围：$(0, 1)$，可以解释为概率
  • 优点：输出有界，适合二分类的输出层
  • 缺点：输出非零中心（均值≈0.5），导致梯度更新效率低；深度网络中容易出现梯度消失（梯度最大值仅 0.25）

2. Tanh 函数

  • 输出范围：$(-1, 1)$，零中心化
  • 优点：输出以0为中心，比Sigmoid收敛更快
  • 缺点：同样存在梯度消失问题（但比Sigmoid轻）

3. ReLU 函数

  • 输出范围：$[0, +\infty)$
  • 优点：计算极快、缓解梯度消失、产生稀疏激活
  • 缺点：Dead ReLU问题 —— 当输入为负时梯度为0，神经元"永久死亡"，不再更新

4. Leaky ReLU 函数

其中 $\alpha$ 是一个小常数（如0.01）。当输入为负时仍保留微小梯度，有效缓解Dead ReLU问题。

5. Softmax 函数

  • 输出范围：$(0, 1)$，所有输出之和为1
  • 适用场景：多分类问题的输出层，将logits转换为概率分布

反向传播算法详解

反向传播是神经网络学习的核心算法。它的本质是微积分中的链式法则（Chain Rule）在计算图上的高效应用。

链式法则

如果 $y = f(g(x))$，则 $\frac{dy}{dx} = \frac{dy}{dg} \cdot \frac{dg}{dx}$。反向传播就是从输出层开始，沿着计算图反向逐层应用链式法则，计算损失函数对每个参数的梯度。

前向传播示例（5步）

以一个简单的两层网络为例，输入 $x=2$，目标输出 $y=1$：

步骤计算结果
① 输入层$x = 2$$a^{(0)} = 2$
② 隐藏层加权求和$z^{(1)} = w_1 \cdot a^{(0)} + b_1 = 0.5 \times 2 + 0.1$$z^{(1)} = 1.1$
③ 隐藏层激活$a^{(1)} = \text{ReLU}(1.1)$$a^{(1)} = 1.1$
④ 输出层加权求和$z^{(2)} = w_2 \cdot a^{(1)} + b_2 = 0.3 \times 1.1 + 0.05$$z^{(2)} = 0.38$
⑤ 输出层激活$\hat{y} = \text{ReLU}(0.38)$$\hat{y} = 0.38$

反向传播梯度计算（8步）

损失函数 $L = \frac{1}{2}(y - \hat{y})^2 = \frac{1}{2}(1 - 0.38)^2 = 0.1922$

步骤计算结果
① $\frac{\partial L}{\partial \hat{y}}$$\hat{y} - y$$-0.62$
② $\frac{\partial \hat{y}}{\partial z^{(2)}}$$\text{ReLU}'(0.38)$$1$
③ $\frac{\partial L}{\partial w_2}$$\frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial z^{(2)}} \cdot a^{(1)}$$-0.682$
④ $\frac{\partial L}{\partial b_2}$$\frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial z^{(2)}}$$-0.62$
⑤ $\frac{\partial L}{\partial a^{(1)}}$$\frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial z^{(2)}} \cdot w_2$$-0.186$
⑥ $\frac{\partial L}{\partial z^{(1)}}$$\frac{\partial L}{\partial a^{(1)}} \cdot \text{ReLU}'(1.1)$$-0.186$
⑦ $\frac{\partial L}{\partial w_1}$$\frac{\partial L}{\partial z^{(1)}} \cdot a^{(0)}$$-0.372$
⑧ $\frac{\partial L}{\partial b_1}$$\frac{\partial L}{\partial z^{(1)}}$$-0.186$

得到所有梯度后，使用梯度下降更新参数：$w \leftarrow w - \alpha \frac{\partial L}{\partial w}$。

import numpy as np

class NeuralNetwork:
    """一个完整的两层神经网络，含前向传播和反向传播"""
    def __init__(self, input_size, hidden_size, output_size, lr=0.01):
        self.lr = lr
        # He初始化
        self.W1 = np.random.randn(input_size, hidden_size) * np.sqrt(2 / input_size)
        self.b1 = np.zeros((1, hidden_size))
        self.W2 = np.random.randn(hidden_size, output_size) * np.sqrt(2 / hidden_size)
        self.b2 = np.zeros((1, output_size))

    def relu(self, z):
        return np.maximum(0, z)

    def relu_derivative(self, z):
        return (z > 0).astype(float)

    def forward(self, X):
        """前向传播"""
        self.z1 = X.dot(self.W1) + self.b1
        self.a1 = self.relu(self.z1)
        self.z2 = self.a1.dot(self.W2) + self.b2
        self.a2 = self.z2  # 回归任务直接输出
        return self.a2

    def backward(self, X, y):
        """反向传播，计算梯度并更新参数"""
        m = X.shape[0]

        # 输出层梯度
        dz2 = self.a2 - y
        dW2 = (1 / m) * self.a1.T.dot(dz2)
        db2 = (1 / m) * np.sum(dz2, axis=0, keepdims=True)

        # 隐藏层梯度
        dz1 = dz2.dot(self.W2.T) * self.relu_derivative(self.z1)
        dW1 = (1 / m) * X.T.dot(dz1)
        db1 = (1 / m) * np.sum(dz1, axis=0, keepdims=True)

        # 梯度下降更新
        self.W2 -= self.lr * dW2
        self.b2 -= self.lr * db2
        self.W1 -= self.lr * dW1
        self.b1 -= self.lr * db1

    def train(self, X, y, epochs=1000):
        for epoch in range(epochs):
            pred = self.forward(X)
            self.backward(X, y)
            if epoch % 200 == 0:
                loss = np.mean((pred - y) ** 2)
                print(f"Epoch {epoch}, Loss: {loss:.6f}")

正则化技术

正则化用于防止过拟合 —— 模型在训练集上表现好但在测试集上表现差。以下是三种最常用的正则化技术：

1. Dropout —— 随机丢弃神经元

训练时以概率 $p$ 随机将部分神经元输出置零，迫使网络学习冗余表示，避免对单个神经元的过度依赖。推理时不使用Dropout。

import torch.nn as nn

            nn.ReLU(),
            nn.Dropout(p=0.5),       # 50%的神经元随机置零
            nn.Linear(256, 128),
            nn.ReLU(),
            nn.Dropout(p=0.3),
            nn.Linear(128, 10)

    def forward(self, x):
        return self.network(x)

滑动的小窗口找猫耳朵 —— 卷积是什么？

想象你在一张巨大的照片中找猫耳朵。你不会一次看整张照片，而是拿着一个小方框（比如3×3像素），在照片上从左到右、从上到下滑动，每次只看一小块区域。

这就是卷积操作 —— 用一个小的"滤波器"（卷积核）在图像上滑动，提取局部特征。

卷积操作的伪代码

# 卷积操作伪代码
def conv2d(image, kernel):
    """
    image: 输入图像 (H x W)
    kernel: 卷积核 (K x K)
    """
    H, W = image.shape
    K = kernel.shape[0]
    output = zeros((H-K+1, W-K+1))

    # 滑动窗口
    for i in range(H - K + 1):
        for j in range(W - K + 1):
            # 取出当前窗口
            window = image[i:i+K, j:j+K]
            # 逐元素相乘再求和
            output[i, j] = sum(window * kernel)

    return output

1. 标准卷积

  • 增加非线性：配合激活函数引入非线性变换

残差学习的核心公式：

网络不再学习完整的映射 H(x)，而是学习残差 F(x) = H(x) - x。通过跳跃连接（skip connection），即使某一层学不到有用信息，至少能恒等映射。

class ResidualBlock(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.conv1 = nn.Conv2d(channels, channels, 3, padding=1)
        self.conv2 = nn.Conv2d(channels, channels, 3, padding=1)
        self.relu = nn.ReLU()

    def forward(self, x):
        residual = x                # 保存输入
        out = self.relu(self.conv1(x))
        out = self.conv2(out)
        out = out + residual        # 残差连接：输出 = F(x) + x
        return self.relu(out)

为什么要除以 $\sqrt{d_k}$？

假设Q和K的每个元素都是独立的随机变量，均值为0，方差为1。那么Q和K的点积的方差为 $d_k$：

当 $d_k$ 很大时，点积的值会很大，导致softmax输出接近one-hot分布（梯度消失）。除以 $\sqrt{d_k}$ 将方差归一化为1，使softmax的输入保持在合理范围。

Query（查询）、Key（键）、Value（值）的直觉理解：

概念含义类比
Query (Q)当前词想要查找的信息搜索引擎中输入的关键词
Key (K)每个词提供的"索引"信息网页的标题和摘要
Value (V)每个词的实际内容网页的正文内容

import torch
import torch.nn as nn
import torch.nn.functional as F

class ScaledDotProductAttention(nn.Module):
    def __init__(self, d_k):
        super().__init__()
        self.scale = d_k ** 0.5

    def forward(self, Q, K, V, mask=None):
        # Q, K, V: (batch, heads, seq_len, d_k)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / self.scale
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        attn_weights = F.softmax(scores, dim=-1)
        output = torch.matmul(attn_weights, V)
        return output, attn_weights

多头注意力机制

多头注意力让模型从不同角度关注信息。将Q、K、V分成h个"头"，每个头独立计算注意力，再拼接起来：

其中每个头的计算为：

$W_Q^i, W_K^i, W_V^i$ 是第i个头的投影矩阵，$W^O$ 是输出投影矩阵。

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        assert d_model % n_heads == 0
        self.d_k = d_model // n_heads
        self.n_heads = n_heads

        self.W_Q = nn.Linear(d_model, d_model)
        self.W_K = nn.Linear(d_model, d_model)
        self.W_V = nn.Linear(d_model, d_model)
        self.W_O = nn.Linear(d_model, d_model)

    def forward(self, Q, K, V, mask=None):
        batch_size = Q.size(0)

        # 线性变换并分头: (batch, seq_len, d_model) -> (batch, n_heads, seq_len, d_k)
        Q = self.W_Q(Q).view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)
        K = self.W_K(K).view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)
        V = self.W_V(V).view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)

        # 缩放点积注意力
        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.d_k ** 0.5)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        attn = F.softmax(scores, dim=-1)
        context = torch.matmul(attn, V)

        # 拼接所有头: (batch, n_heads, seq_len, d_k) -> (batch, seq_len, d_model)
        context = context.transpose(1, 2).contiguous().view(batch_size, -1, self.n_heads * self.d_k)

        # 输出投影
        output = self.W_O(context)
        return output

位置编码

Transformer没有循环结构，需要显式注入位置信息。原始论文使用正弦/余弦位置编码：

其中 $pos$ 是位置，$i$ 是维度索引，$d_{model}$ 是模型维度。偶数维度用正弦，奇数维度用余弦。

这种编码的优点：对于任意固定偏移 $k$，$PE(pos+k)$ 可以表示为 $PE(pos)$ 的线性函数，因此模型可以学习相对位置关系。

import torch
import torch.nn as nn
import math

class PositionalEncoding(nn.Module):
    def __init__(self, d_model, max_len=5000):
        super().__init__()
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len).unsqueeze(1).float()
        div_term = torch.exp(torch.arange(0, d_model, 2).float() * -(math.log(10000.0) / d_model))

        pe[:, 0::2] = torch.sin(position * div_term)  # 偶数维度
        pe[:, 1::2] = torch.cos(position * div_term)  # 奇数维度

        pe = pe.unsqueeze(0)  # (1, max_len, d_model)
        self.register_buffer('pe', pe)

    def forward(self, x):
        # x: (batch, seq_len, d_model)
        return x + self.pe[:, :x.size(1), :]

Transformer编码器完整结构

一个Transformer编码器层包含：多头自注意力 → 残差连接+层归一化 → 前馈网络 → 残差连接+层归一化。

class TransformerEncoderLayer(nn.Module):
    def __init__(self, d_model, n_heads, d_ff, dropout=0.1):
        super().__init__()
        self.self_attn = MultiHeadAttention(d_model, n_heads)
        self.feed_forward = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.ReLU(),
            nn.Linear(d_ff, d_model)
        )
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x, mask=None):
        # 多头自注意力 + 残差连接 + 层归一化
        attn_output = self.self_attn(x, x, x, mask)
        x = self.norm1(x + self.dropout(attn_output))

        # 前馈网络 + 残差连接 + 层归一化
        ff_output = self.feed_forward(x)
        x = self.norm2(x + self.dropout(ff_output))
        return x

class TransformerEncoder(nn.Module):
    def __init__(self, vocab_size, d_model, n_heads, d_ff, n_layers, max_len=5000):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, d_model)
        self.pos_encoding = PositionalEncoding(d_model, max_len)
        self.layers = nn.ModuleList([
            TransformerEncoderLayer(d_model, n_heads, d_ff)
            for _ in range(n_layers)
        ])
        self.d_model = d_model

    def forward(self, x, mask=None):
        # 词嵌入 + 位置编码，并缩放
        x = self.embedding(x) * math.sqrt(self.d_model)
        x = self.pos_encoding(x)
        for layer in self.layers:
            x = layer(x, mask)
        return x

K-Means聚类 —— 最简单的聚类算法

生活比喻：想象你是一个学校食堂管理员，需要决定开几个打饭窗口。你会观察学生们的排队习惯，自动把他们分成几组。

K-Means的工作步骤：

  1. 初始化：随机选择K个数据点作为初始中心
  2. 分配：将每个数据点分配到最近的中心点
  3. 更新：重新计算每个簇的中心点
  4. 重复：直到中心点不再变化或达到最大迭代次数

K-Means的数学表达：

K-Means的目标是最小化簇内平方误差（Within-Cluster Sum of Squares, WCSS）：

其中 $\mu_k$ 是第 $k$ 个簇的中心点，$r_{nk}$ 是指示变量（$r_{nk}=1$ 表示点 $n$ 属于簇 $k$）。

K-Means的收敛性：

定理：K-Means算法在每次迭代中都会使目标函数 $J$ 单调递减，因此保证收敛。但由于目标函数是非凸的，算法可能收敛到局部最优解。

收敛过程保证：

  • 分配步骤：每个点分配到最近的中心，使 $J$ 减小或不变
  • 更新步骤：中心点移向簇的均值，使 $J$ 减小或不变
  • $J$ 有下界（$\geq 0$），且每次迭代严格递减

因此，K-Means一定在有限步内收敛。通常10-30次迭代即可收敛。

DBSCAN —— 基于密度的聚类

K-Means的局限性：只能发现球形簇，对噪声敏感。DBSCAN（Density-Based Spatial Clustering of Applications with Noise）解决了这些问题。

核心概念：

概念定义生活类比
ε-邻域以点为中心、半径为ε的圆形区域你家周围的"步行可达"范围
MinPts成为核心点所需的最少邻居数开饭馆需要的最少顾客数

噪声点既不是核心点也不是边界点的点荒郊野外的孤零零房子

DBSCAN的工作流程：

  1. 选择任意一个未访问的点
  2. 计算其ε-邻域内的点数
  3. 如果是核心点（≥MinPts），创建一个新簇
  4. 从该核心点出发，将所有密度可达的点加入同一簇
  5. 如果遇到边界点，将其加入当前簇但不再向外扩展
  6. 重复以上步骤，直到所有点都被访问

PCA降维 —— 数学推导

生活比喻：你在描述一个人时，可以说身高、体重、年龄、学历、收入等100个特征。但其实核心可能就是"经济实力"和"健康状况"两个维度。PCA就是帮你找到最重要的维度。

PCA的核心思想：找到数据变化最大的方向，保留最重要的信息，去掉冗余。

PCA的数学推导（特征值分解）：

给定 $n$ 个 $d$ 维样本 $\mathbf{x}_1, \mathbf{x}_2, \ldots, \mathbf{x}_n$，PCA的目标是找到 $k$ 个方向（$k < d$），使数据投影后的方差最大。

步骤1：中心化数据

步骤2：计算协方差矩阵

步骤3：特征值分解

步骤4：选择主成分

对特征值从大到小排序 $\lambda_1 \geq \lambda_2 \geq \cdots \geq \lambda_d$，选择前 $k$ 个特征向量，使得累积方差贡献率达到阈值（如95%）：

步骤5：投影

# 手动实现PCA（理解原理）
def my_pca(X, n_components):
    # 1. 中心化
    X_centered = X - np.mean(X, axis=0)
    # 2. 计算协方差矩阵
    cov = np.cov(X_centered.T)
    # 3. 特征值分解
    eigenvalues, eigenvectors = np.linalg.eig(cov)
    # 4. 按特征值排序
    idx = np.argsort(eigenvalues)[::-1]
    eigenvalues = eigenvalues[idx]
    eigenvectors = eigenvectors[:, idx]
    # 5. 选择前k个特征向量
    W = eigenvectors[:, :n_components]
    # 6. 投影
    X_projected = X_centered @ W
    return X_projected, eigenvalues

X_proj, eigenvals = my_pca(X, n_components=3)
print(f"\n手动PCA结果形状: {X_proj.shape}")

import numpy as np

class QLearningAgent:
    def __init__(self, n_states, n_actions, lr=0.1, gamma=0.99, epsilon=0.1):
        self.q_table = np.zeros((n_states, n_actions))
        self.lr = lr
        self.gamma = gamma
        self.epsilon = epsilon
        self.n_actions = n_actions

    def choose_action(self, state):
        """ε-贪心策略"""
        if np.random.random() < self.epsilon:
            return np.random.randint(self.n_actions)
        return np.argmax(self.q_table[state])

    def update(self, state, action, reward, next_state, done):
        """Q-Learning更新公式"""
        if done:
            target = reward
        else:
            target = reward + self.gamma * np.max(self.q_table[next_state])

        # Q(s,a) ← Q(s,a) + α[r + γ·max Q(s',a') - Q(s,a)]
        self.q_table[state, action] += self.lr * (target - self.q_table[state, action])

    def train(self, env, episodes=1000):
        rewards_per_episode = []
        for episode in range(episodes):
            state = env.reset()
            total_reward = 0
            done = False

            while not done:
                action = self.choose_action(state)
                next_state, reward, done = env.step(action)
                self.update(state, action, reward, next_state, done)
                state = next_state
                total_reward += reward

            rewards_per_episode.append(total_reward)
            # 探索率衰减
            self.epsilon = max(0.01, self.epsilon * 0.995)

        return rewards_per_episode

import torch
import torch.nn as nn
import numpy as np
from collections import deque
import random

class DQN(nn.Module):
    """DQN网络：输入状态，输出每个动作的Q值"""
    def __init__(self, state_dim, action_dim):
        super().__init__()
        self.network = nn.Sequential(
            nn.Linear(state_dim, 128),
            nn.ReLU(),
            nn.Linear(128, 128),
            nn.ReLU(),
            nn.Linear(128, action_dim)
        )

    def forward(self, x):
        return self.network(x)

class DQNAgent:
    def __init__(self, state_dim, action_dim, lr=1e-3, gamma=0.99,
                 epsilon=1.0, epsilon_min=0.01, buffer_size=10000, batch_size=64):
        self.action_dim = action_dim
        self.gamma = gamma
        self.epsilon = epsilon
        self.epsilon_min = epsilon_min
        self.batch_size = batch_size

        # 在线网络和目标网络
        self.q_net = DQN(state_dim, action_dim)
        self.target_net = DQN(state_dim, action_dim)
        self.target_net.load_state_dict(self.q_net.state_dict())

        self.optimizer = torch.optim.Adam(self.q_net.parameters(), lr=lr)
        self.loss_fn = nn.MSELoss()

        # 经验回放缓冲区
        self.replay_buffer = deque(maxlen=buffer_size)

    def choose_action(self, state):
        if random.random() < self.epsilon:
            return random.randint(0, self.action_dim - 1)
        with torch.no_grad():
            state_t = torch.FloatTensor(state).unsqueeze(0)
            q_values = self.q_net(state_t)
            return q_values.argmax(dim=1).item()

    def store(self, state, action, reward, next_state, done):
        self.replay_buffer.append((state, action, reward, next_state, done))

    def train_step(self):
        if len(self.replay_buffer) < self.batch_size:
            return

        # 从缓冲区随机采样
        batch = random.sample(self.replay_buffer, self.batch_size)
        states, actions, rewards, next_states, dones = zip(*batch)

        states = torch.FloatTensor(np.array(states))
        actions = torch.LongTensor(actions).unsqueeze(1)
        rewards = torch.FloatTensor(rewards)
        next_states = torch.FloatTensor(np.array(next_states))
        dones = torch.FloatTensor(dones)

        # 计算当前Q值
        q_values = self.q_net(states).gather(1, actions).squeeze()

        # 计算目标Q值（使用目标网络）
        with torch.no_grad():
            next_q_values = self.target_net(next_states).max(dim=1)[0]
            targets = rewards + self.gamma * next_q_values * (1 - dones)

        # 更新在线网络
        loss = self.loss_fn(q_values, targets)
        self.optimizer.zero_grad()
        loss.backward()
        self.optimizer.step()

        return loss.item()

    def update_target(self):
        """定期将在线网络参数复制到目标网络"""
        self.target_net.load_state_dict(self.q_net.state_dict())

策略梯度方法 —— 直接优化策略

Q-Learning的问题：它学习的是价值函数，然后间接得到策略。策略梯度方法直接优化策略 $\pi_\theta(a|s)$，参数化为神经网络。

策略梯度定理：

直觉理解：如果采取某个动作使回报增加，就增大这个动作的概率；反之减小。

REINFORCE算法：

import torch
import torch.nn as nn
import torch.nn.functional as F

class PolicyNetwork(nn.Module):
    def __init__(self, state_dim, action_dim):
        super().__init__()
        self.network = nn.Sequential(
            nn.Linear(state_dim, 128),
            nn.ReLU(),
            nn.Linear(128, action_dim),
            nn.Softmax(dim=-1)
        )

    def forward(self, x):
        return self.network(x)

def REINFORCE(env, policy, lr=1e-3, gamma=0.99):
    optimizer = torch.optim.Adam(policy.parameters(), lr=lr)

    for episode in range(1000):
        state = env.reset()
        log_probs = []
        rewards = []

        # 收集轨迹
        done = False
        while not done:
            state_t = torch.FloatTensor(state).unsqueeze(0)
            probs = policy(state_t)
            action_dist = torch.distributions.Categorical(probs)
            action = action_dist.sample()

            log_probs.append(action_dist.log_prob(action))
            state, reward, done = env.step(action.item())
            rewards.append(reward)

        # 计算折扣回报
        returns = []
        G = 0
        for r in reversed(rewards):
            G = r + gamma * G
            returns.insert(0, G)
        returns = torch.FloatTensor(returns)
        returns = (returns - returns.mean()) / (returns.std() + 1e-8)

        # 策略梯度更新
        loss = 0
        for log_prob, G in zip(log_probs, returns):
            loss -= log_prob * G

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

Attention机制 —— 数学推导

Attention机制的核心思想：让模型在处理每个位置时，能"关注"输入序列中最相关的部分。

缩放点积注意力（Scaled Dot-Product Attention）：

其中 $Q$（查询）、$K$（键）、$V$（值）都是输入通过线性变换得到的，$d_k$ 是键向量维度。

为什么要除以 $\sqrt{d_k}$？

假设 $Q$ 和 $K$ 的元素独立同分布，均值为0，方差为1。则点积 $Q \cdot K$ 的方差为 $d_k$。当 $d_k$ 很大时，softmax的输入值会很大，导致梯度消失。除以 $\sqrt{d_k}$ 将方差归一化为1。

Self-Attention的完整流程：

  1. 输入序列 $\mathbf{X} \in \mathbb{R}^{n \times d}$
  2. 计算 $Q = \mathbf{X}W_Q$, $K = \mathbf{X}W_K$, $V = \mathbf{X}W_V$
  3. 计算注意力分数 $\text{score} = \frac{QK^T}{\sqrt{d_k}}$
  4. Softmax归一化得到注意力权重
  5. 对 $V$ 加权求和得到输出

给照片化妆 —— GAN的魔力

想象一个画家和一个鉴定师在博弈：画家不断练习画得更逼真，鉴定师不断学习分辨真假。最终画家能画出以假乱真的作品。

GAN（生成对抗网络）：由生成器（画家）和判别器（鉴定师）对抗训练，生成器学会创造逼真的数据。

GAN的数学原理

GAN由两个网络组成：生成器 $G$ 和判别器 $D$。它们通过对抗训练共同进步。

GAN的损失函数（min-max目标）：

其中 $x$ 是真实数据，$z$ 是随机噪声，$G(z)$ 是生成器生成的假数据。

训练过程：

  • 判别器 $D$ 的目标：最大化 $\log D(x) + \log(1 - D(G(z)))$，即正确区分真假
  • 生成器 $G$ 的目标：最小化 $\log(1 - D(G(z)))$，即让判别器分不清真假

训练过程的直觉理解：

第一步（固定G，训练D）：给判别器看真图和假图，让它学会区分。判别器越好，生成器越有动力改进。

第二步（固定D，训练G）：生成器生成假图，骗判别器。如果判别器被骗了，说明生成器进步了。

收敛条件：当生成器生成的数据分布与真实数据分布完全一致时，判别器的输出概率为0.5（无法区分）。

import torch
import torch.nn as nn

class Generator(nn.Module):
    """生成器：将噪声z映射为假图像"""
    def __init__(self, latent_dim=100, img_dim=784):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(latent_dim, 256),
            nn.LeakyReLU(0.2),
            nn.Linear(256, 512),
            nn.LeakyReLU(0.2),
            nn.Linear(512, img_dim),
            nn.Tanh()  # 输出范围[-1, 1]
        )

    def forward(self, z):
        return self.net(z)

class Discriminator(nn.Module):
    """判别器：判断输入是真还是假"""
    def __init__(self, img_dim=784):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(img_dim, 512),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.3),
            nn.Linear(512, 256),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.3),
            nn.Linear(256, 1),
            nn.Sigmoid()
        )

    def forward(self, x):
        return self.net(x)

def train_gan(generator, discriminator, dataloader, epochs=100):
    criterion = nn.BCELoss()
    opt_G = torch.optim.Adam(generator.parameters(), lr=2e-4, betas=(0.5, 0.999))
    opt_D = torch.optim.Adam(discriminator.parameters(), lr=2e-4, betas=(0.5, 0.999))

    for epoch in range(epochs):
        for real_imgs, _ in dataloader:
            batch_size = real_imgs.size(0)
            real_imgs = real_imgs.view(batch_size, -1)

            # 真实标签和假标签
            real_labels = torch.ones(batch_size, 1)
            fake_labels = torch.zeros(batch_size, 1)

            # === 训练判别器 ===
            z = torch.randn(batch_size, 100)  # 随机噪声
            fake_imgs = generator(z)

            d_real = discriminator(real_imgs)
            d_fake = discriminator(fake_imgs.detach())  # detach阻止梯度传到G

            d_loss = criterion(d_real, real_labels) + criterion(d_fake, fake_labels)

            opt_D.zero_grad()
            d_loss.backward()
            opt_D.step()

            # === 训练生成器 ===
            z = torch.randn(batch_size, 100)
            fake_imgs = generator(z)
            d_fake = discriminator(fake_imgs)

            g_loss = criterion(d_fake, real_labels)  # G希望D认为是真

            opt_G.zero_grad()
            g_loss.backward()
            opt_G.step()

YOLO（You Only Look Once）把目标检测变成一个回归问题：输入一张图，直接输出所有物体的位置和类别。

YOLO的核心思想：

  1. 网格划分：将图像划分为 $S \times S$ 的网格
  2. 边界框预测：每个网格预测 $B$ 个边界框（每个框包含 $x, y, w, h$ 和置信度）

Vision Transformer（ViT）—— Transformer做视觉

核心思想：将图像分成固定大小的块（patch），每个块当作一个"词"，用Transformer处理。

图像 → 切块 → 展平 → 加位置编码 → Transformer编码器 → 分类

224×224图像 → 16×16=196个patch → 每个patch是一个token

import torch
import torch.nn as nn

class PatchEmbedding(nn.Module):
    """将图像切分为patch并嵌入"""
    def __init__(self, img_size=224, patch_size=16, in_channels=3, embed_dim=768):
        super().__init__()
        self.patch_size = patch_size
        self.n_patches = (img_size // patch_size) ** 2  # 196

        # 用卷积实现切块+嵌入
        self.proj = nn.Conv2d(in_channels, embed_dim,
                              kernel_size=patch_size, stride=patch_size)

    def forward(self, x):
        # x: (batch, 3, 224, 224)
        x = self.proj(x)         # (batch, 768, 14, 14)
        x = x.flatten(2).transpose(1, 2)  # (batch, 196, 768)
        return x

class VisionTransformer(nn.Module):
    def __init__(self, img_size=224, patch_size=16, n_classes=1000,
                 embed_dim=768, depth=12, n_heads=12):
        super().__init__()
        self.patch_embed = PatchEmbedding(img_size, patch_size, embed_dim=embed_dim)

        # CLS token和位置编码
        self.cls_token = nn.Parameter(torch.randn(1, 1, embed_dim))
        self.pos_embed = nn.Parameter(torch.randn(1, self.patch_embed.n_patches + 1, embed_dim))

        # Transformer编码器
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=embed_dim, nhead=n_heads, dim_feedforward=embed_dim*4
        )
        self.transformer = nn.TransformerEncoder(encoder_layer, num_layers=depth)

        # 分类头
        self.head = nn.Linear(embed_dim, n_classes)

    def forward(self, x):
        batch_size = x.shape[0]
        x = self.patch_embed(x)  # (batch, 196, 768)

        # 添加CLS token
        cls_tokens = self.cls_token.expand(batch_size, -1, -1)
        x = torch.cat([cls_tokens, x], dim=1)  # (batch, 197, 768)

        # 添加位置编码
        x = x + self.pos_embed

        # Transformer编码
        x = x.transpose(0, 1)  # (seq_len, batch, embed_dim)
        x = self.transformer(x)

        # 用CLS token做分类
        x = x[0]  # 取CLS token
        x = self.head(x)
        return x

# ViT-B/16: 86M参数, patch_size=16, embed_dim=768, depth=12
model = VisionTransformer(img_size=224, patch_size=16, n_classes=1000)

AI的道德课 —— 技术向善

算法公平性 —— AI也会"歧视"

真实案例：亚马逊的AI招聘系统被发现歧视女性候选人。原因是训练数据中男性简历更多，AI学会了"男性更适合"的偏见。

偏见来源：数据偏见 → 模型学习偏见 → 放大偏见 → 造成不公平

公平性的数学定义：

1. 统计均等（Demographic Parity）：

要求：无论受保护属性（如性别、种族）如何，获得正类预测的概率应相同。

2. 均等机会（Equalized Odds）：

要求：不同群体的真正率和假正率相同。即对不同群体的预测准确率应该一致。

3. 校准（Calibration）：

要求：预测概率为 $p$ 的样本中，真正属于正类的比例确实是 $p$。

公平性定义的不可能定理：

当受保护属性与标签相关时，统计均等、均等机会和校准不可能同时满足。我们必须根据具体应用场景选择最合适的公平性定义。

生活比喻：如果你只用左眼看世界，你会觉得所有东西都偏向左边。AI如果只看有偏见的数据，也会产生偏见。

差分隐私 —— 数学化的隐私保护

问题：机器学习需要大量数据，但数据包含个人隐私。如何在训练模型的同时保护隐私？

差分隐私（Differential Privacy）的定义：

一个随机算法 $M$ 满足 $(\varepsilon, \delta)$-差分隐私，当且仅当对于任意两个相邻数据集 $D_1$ 和 $D_2$（仅差一条记录），以及任意输出子集 $S$：

其中 $\varepsilon$（隐私预算）越小，隐私保护越强。$\varepsilon=0$ 表示完全隐私，$\varepsilon=\infty$ 表示无隐私保护。

噪声注入机制：

最常用的实现方式是在梯度上添加噪声（差分隐私随机梯度下降，DP-SGD）：

其中 $\sigma$ 是噪声标准差，$\sigma = \frac{S \cdot \sqrt{2 \ln(1.25/\delta)}}{\varepsilon}$，$S$ 是梯度裁剪阈值。

import torch
import numpy as np

def dp_sgd_step(model, dataloader, lr, noise_multiplier, max_grad_norm):
    """差分隐私SGD的一步更新"""
    for p in model.parameters():
        if p.grad is not None:
            # 1. 梯度裁剪（按样本裁剪）
            grad_norm = torch.norm(p.grad, p=2)
            clip_factor = max_grad_norm / (grad_norm + 1e-8)
            if clip_factor < 1:
                p.grad *= clip_factor

            # 2. 添加高斯噪声
            noise = torch.randn_like(p.grad) * noise_multiplier * max_grad_norm
            p.grad += noise

            # 3. 标准SGD更新
            p.data -= lr * p.grad

联邦学习 —— 数据不出本地的AI

核心思想：数据保留在本地设备上，只共享模型参数更新，而不是原始数据。

  2. 各客户端在本地数据上训练
  3. 客户端只上传模型更新（梯度）
  4. 服务器聚合更新，生成新的全局模型
  5. 重复以上步骤直到收敛

            f"1. ..."
        )

        """行动：执行步骤"""
        if step.startswith("调用工具:"):
            tool_name = step.split(":")[1]
            result = self.tools[tool_name].run()
            self.memory.append({"role": "tool", "content": result})
            return result
        elif step.startswith("生成回复:"):
            return self.llm.generate(step)
        else:
            return self.execute_code(step)

        """完整的执行循环"""
        plan = self.think(task)
        results = []
        for step in plan:
            result = self.act(step)
            results.append(result)
            if not self.verify(result, task):
                plan = self.revise_plan(plan, result)
        return results
