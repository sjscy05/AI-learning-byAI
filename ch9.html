📝 第九章：自然语言处理进阶

给词语画地图 —— 词嵌入

计算机不理解"国王"、"女王"这些词的含义。但如果我们把每个词映射到一个向量空间，让意思相近的词距离也近，计算机就能"理解"词语了。

词嵌入(Word Embedding)：将词语映射为稠密向量，使得语义相近的词在向量空间中距离也相近。

生活比喻：想象一个巨大的地图，每个词是一个城市。"北京"和"上海"挨得很近（都是中国大城市），"猫"和"狗"挨得很近（都是宠物），但"北京"和"猫"就离得很远。

Word2Vec —— 两种训练方式

Word2Vec是Google在2013年提出的词嵌入方法，有两种训练架构：

1. CBOW（Continuous Bag of Words）—— 根据上下文预测中心词

输入（上下文） → 预测 → 输出（中心词）

"the cat ___ on the mat" → 预测 "sat"

适合：频繁词、语法规则学习

2. Skip-gram —— 根据中心词预测上下文

输入（中心词） → 预测 → 输出（上下文）

"sat" → 预测 "the", "cat", "on", "the", "mat"

适合：稀疏词、语义关系学习

CBOW vs Skip-gram 对比：

特性CBOWSkip-gram
训练速度快较慢
罕见词效果一般更好
语义相似度较好更好
语法规则更好一般

Attention机制 —— 数学推导

Attention机制的核心思想：让模型在处理每个位置时，能"关注"输入序列中最相关的部分。

缩放点积注意力（Scaled Dot-Product Attention）：

其中 $Q$（查询）、$K$（键）、$V$（值）都是输入通过线性变换得到的，$d_k$ 是键向量维度。

为什么要除以 $\sqrt{d_k}$？

假设 $Q$ 和 $K$ 的元素独立同分布，均值为0，方差为1。则点积 $Q \cdot K$ 的方差为 $d_k$。当 $d_k$ 很大时，softmax的输入值会很大，导致梯度消失。除以 $\sqrt{d_k}$ 将方差归一化为1。

Self-Attention的完整流程：

  1. 输入序列 $\mathbf{X} \in \mathbb{R}^{n \times d}$
  2. 计算 $Q = \mathbf{X}W_Q$, $K = \mathbf{X}W_K$, $V = \mathbf{X}W_V$
  3. 计算注意力分数 $\text{score} = \frac{QK^T}{\sqrt{d_k}}$
  4. Softmax归一化得到注意力权重
  5. 对 $V$ 加权求和得到输出

BERT —— 双向预训练模型

BERT（Bidirectional Encoder Representations from Transformers）是Google在2018年提出的预训练语言模型。核心创新：双向上下文理解。

预训练任务1：掩码语言模型（MLM）

随机遮盖15%的词，让模型预测被遮盖的词：

输入: "我 喜欢 [MASK] 算法"
标签: "机器"

80%替换为[MASK]，10%替换为随机词，10%保持不变。

预训练任务2：下一句预测（NSP）

判断两个句子是否是连续的：

输入: "[CLS] 今天天气很好 [SEP] 适合出去玩 [SEP]" → IsNext
输入: "[CLS] 今天天气很好 [SEP] 苹果是一种水果 [SEP]" → NotNext

BERT微调示例：

from transformers import BertTokenizer, BertForSequenceClassification

# 准备数据

# 前向传播
outputs = model(**inputs, labels=torch.tensor(labels))

training_args = TrainingArguments(

GPT —— 自回归生成

GPT（Generative Pre-trained Transformer）使用Transformer解码器，通过自回归方式生成文本。

GPT的训练目标：给定前文，预测下一个词（语言建模）：

其中 $\theta$ 是模型参数，$u_i$ 是第 $i$ 个词的上下文。

BERT vs GPT 对比：

特性BERTGPT
架构Transformer编码器Transformer解码器
注意力方向双向（同时看左右）单向（只看左边）
预训练任务MLM + NSP语言模型（预测下一个词）
适合任务理解任务（分类、NER）生成任务（对话、写作）
参数量1.1亿（base）1750亿（GPT-3）
 
# GPT文本生成示例
from transformers import GPT2LMHeadModel, GPT2Tokenizer

tokenizer = GPT2Tokenizer.from_pretrained('gpt2')
model = GPT2LMHeadModel.from_pretrained('gpt2')

# 自回归生成
def generate_text(prompt, max_length=50, temperature=0.8):
    inputs = tokenizer.encode(prompt, return_tensors='pt')

    outputs = model.generate(
        inputs,
        max_length=max_length,
        temperature=temperature,  # 控制随机性
        top_k=50,                 # Top-K采样
        top_p=0.95,               # Nucleus采样
        do_sample=True,
        num_return_sequences=1
    )
    return tokenizer.decode(outputs[0], skip_special_tokens=True)

# 生成文本
prompt = "The future of AI is"
generated = generate_text(prompt)
print(generated)

RNN与LSTM —— 处理序列数据

问题：普通神经网络处理固定长度的输入。但句子有长有短。

解决方案：RNN（循环神经网络）有一个"记忆"，可以记住前面看过的内容。

LSTM的比喻 —— 带选择的记忆：

  • 遗忘门：决定丢掉哪些旧记忆（"昨天吃了什么不重要"）
  • 输入门：决定记住哪些新信息（"今天学了Python很重要"）
  • 输出门：决定输出什么（"现在需要回忆Python知识"）
# 文本情感分析示例（简化版）
from transformers import pipeline

# 加载预训练的情感分析模型
classifier = pipeline("sentiment-analysis")

# 分析电影评论的情感
reviews = [
    "这部电影太精彩了，强烈推荐！",
    "剧情拖沓，浪费了两个小时。",
    "演员演技在线，值得一看。"
]

for review in reviews:
    result = classifier(review)[0]
    print(f"评论: {review}")
    print(f"情感: {result['label']}, 置信度: {result['score']:.2f}")
    print()

练习题

练习1：训练一个Word2Vec模型（使用gensim），在中文语料上学习词向量，验证"国王-男人+女人≈女王"的语义关系。

💡 指引： Word2Vec通过上下文预测词（CBOW）或通过词预测上下文（Skip-gram）来学习词向量。语义关系验证利用了词向量空间的线性性质——向量运算能捕获语义关系。注意：中文需要先分词，语料量越大、语义关系越准确。

练习2：解释Self-Attention中为什么要除以 $\sqrt{d_k}$。如果不除，会发生什么？

💡 指引： Q和K的点积结果的方差会随维度d_k增大而增大。如果不缩放，softmax会进入梯度极小的饱和区，导致训练困难。除以√d_k使点积的方差保持在合理范围，让softmax的输出更平滑。

练习3：使用BERT对中文文本进行情感分类。比较BERT和传统机器学习方法（如TF-IDF+SVM）的效果差异。

💡 指引： BERT通过预训练获得了深层语义理解能力，能处理一词多义、上下文依赖等TF-IDF无法捕获的语义。TF-IDF是词袋模型，忽略词序和语义。但BERT需要更多计算资源，且在小数据集上微调时要注意过拟合。

练习4：比较BERT和GPT的架构差异，分析为什么BERT更适合理解任务，GPT更适合生成任务。

💡 指引： BERT使用双向Transformer（能看到前后文），适合理解完整句子；GPT使用单向Transformer（只能看前文），天然适合逐词生成。BERT的预训练目标是"完形填空"（理解），GPT是"预测下一个词"（生成）。架构决定了各自的天然优势。
