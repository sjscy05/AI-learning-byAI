⚖️ 第十一章：AI伦理与社会影响

AI的道德课 —— 技术向善

AI技术越来越强大，但强大的技术也需要负责任地使用。就像一把菜刀可以切菜也可以伤人，AI的使用方式决定了它对社会的影响。

算法公平性 —— AI也会"歧视"

真实案例：亚马逊的AI招聘系统被发现歧视女性候选人。原因是训练数据中男性简历更多，AI学会了"男性更适合"的偏见。

偏见来源：数据偏见 → 模型学习偏见 → 放大偏见 → 造成不公平

公平性的数学定义：

1. 统计均等（Demographic Parity）：

要求：无论受保护属性（如性别、种族）如何，获得正类预测的概率应相同。

2. 均等机会（Equalized Odds）：

要求：不同群体的真正率和假正率相同。即对不同群体的预测准确率应该一致。

3. 校准（Calibration）：

要求：预测概率为 $p$ 的样本中，真正属于正类的比例确实是 $p$。

公平性定义的不可能定理：

当受保护属性与标签相关时，统计均等、均等机会和校准不可能同时满足。我们必须根据具体应用场景选择最合适的公平性定义。

生活比喻：如果你只用左眼看世界，你会觉得所有东西都偏向左边。AI如果只看有偏见的数据，也会产生偏见。

差分隐私 —— 数学化的隐私保护

问题：机器学习需要大量数据，但数据包含个人隐私。如何在训练模型的同时保护隐私？

差分隐私（Differential Privacy）的定义：

一个随机算法 $M$ 满足 $(\varepsilon, \delta)$-差分隐私，当且仅当对于任意两个相邻数据集 $D_1$ 和 $D_2$（仅差一条记录），以及任意输出子集 $S$：

其中 $\varepsilon$（隐私预算）越小，隐私保护越强。$\varepsilon=0$ 表示完全隐私，$\varepsilon=\infty$ 表示无隐私保护。

噪声注入机制：

最常用的实现方式是在梯度上添加噪声（差分隐私随机梯度下降，DP-SGD）：

其中 $\sigma$ 是噪声标准差，$\sigma = \frac{S \cdot \sqrt{2 \ln(1.25/\delta)}}{\varepsilon}$，$S$ 是梯度裁剪阈值。

# 差分隐私的简单实现
import torch
import numpy as np

def dp_sgd_step(model, dataloader, lr, noise_multiplier, max_grad_norm):
    """差分隐私SGD的一步更新"""
    for p in model.parameters():
        if p.grad is not None:
            # 1. 梯度裁剪（按样本裁剪）
            grad_norm = torch.norm(p.grad, p=2)
            clip_factor = max_grad_norm / (grad_norm + 1e-8)
            if clip_factor < 1:
                p.grad *= clip_factor

            # 2. 添加高斯噪声
            noise = torch.randn_like(p.grad) * noise_multiplier * max_grad_norm
            p.grad += noise

            # 3. 标准SGD更新
            p.data -= lr * p.grad

# ε = 8 对应较好的隐私保护
# ε = 1 对应非常强的隐私保护
# ε = 0.1 几乎完全隐私保护

联邦学习 —— 数据不出本地的AI

核心思想：数据保留在本地设备上，只共享模型参数更新，而不是原始数据。

联邦学习的流程：

  1. 服务器发送全局模型给各客户端
  2. 各客户端在本地数据上训练
  3. 客户端只上传模型更新（梯度）
  4. 服务器聚合更新，生成新的全局模型
  5. 重复以上步骤直到收敛

服务器 ↔ 手机1（本地数据）

↔ 手机2（本地数据）

↔ 手机3（本地数据）

数据不出手机，只上传模型参数

联邦学习的挑战：

  • 数据异构：不同客户端的数据分布可能差异很大
  • 通信效率：模型更新需要大量通信
  • 安全性：攻击者可能通过梯度推断原始数据

可解释AI —— 让AI决策透明化

问题：深度学习模型是"黑箱"，我们不知道它为什么做出某个决策。在医疗、金融等高风险领域，不可解释的AI是不可接受的。

两种可解释性：

类型定义方法
全局可解释性理解模型整体行为特征重要性、部分依赖图
局部可解释性解释单个预测LIME、SHAP

LIME（Local Interpretable Model-agnostic Explanations）：

在待解释样本的邻域内训练一个简单的可解释模型（如线性模型）来近似黑箱模型。

SHAP（SHapley Additive exPlanations）：

其中 $S$ 是特征子集，$|S|$ 是子集大小，$d$ 是特征总数。

# SHAP可解释性示例
import shap
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_iris

# 训练模型
X, y = load_iris(return_X_y=True, as_frame=True)
model = RandomForestClassifier(n_estimators=100, random_state=42)
model.fit(X, y)

# 计算SHAP值
explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X)

# 全局特征重要性
shap.summary_plot(shap_values, X)

# 单个样本解释
shap.force_plot(explainer.expected_value[0],
                shap_values[0][0],
                X.iloc[0])

# LIME解释
from lime.lime_tabular import LimeTabularExplainer
lime_explainer = LimeTabularExplainer(
    X.values, feature_names=X.columns.tolist(),
    class_names=['setosa', 'versicolor', 'virginica']
)
explanation = lime_explainer.explain_instance(
    X.iloc[0].values, model.predict_proba, num_features=4
)
explanation.show_in_notebook()

思考题

  1. 如果AI法官判案，你会信任它的判决吗？如何定义"公平"的AI法官？

    💡 指引： 思考"公平"的多重定义：统计公平（不同群体判决率相同）、个体公平（相似案件相似判决）、程序公平（决策过程可解释）。不同公平定义之间可能存在数学上的不可兼得。AI法官的优势是一致性，但缺乏人类的同理法和情境判断。

  2. 自动驾驶汽车在紧急情况下应该保护乘客还是行人？不同文化背景下答案会不同吗？

    💡 指引： 这是经典的"电车问题"变体。功利主义倾向于最小化总伤害，义务论可能强调不能主动选择牺牲谁。不同文化确实有不同偏好——集体主义文化可能更接受牺牲少数保护多数，个人主义文化可能更强调个体权利。现实中还需要考虑法律责任和用户接受度。

  3. AI生成的艺术作品算"创作"吗？版权应该归谁？

    💡 指引： 关键在于"创作"的定义——是否需要人类的创造性意图？如果AI只是工具（如Photoshop），版权可能归使用者；如果AI自主生成，版权归属就模糊了。还要考虑训练数据中原始作者的权益——AI是否在"学习"还是"复制"？

  4. 差分隐私的隐私预算 $\varepsilon$ 应该由谁来设定？用户、企业还是政府？

    💡 指引： ε越小隐私保护越强，但数据可用性越低——这是隐私与效用的权衡。用户设定可能缺乏技术理解，企业设定可能偏向数据利用，政府设定可能一刀切。现实中的做法通常是多方协商，结合具体场景（医疗vs广告）设定不同标准。

  5. 联邦学习真的能完全保护隐私吗？可能存在哪些攻击方式？

    💡 指引： 联邦学习保护的是原始数据不离开本地，但梯度/模型更新仍可能泄露信息。攻击方式包括：1）梯度反演攻击（从梯度推断训练数据）；2）成员推断攻击（判断某条数据是否参与训练）；3）模型投毒攻击。通常需要结合差分隐私和安全聚合来增强保护。
